# Flexo Print Website

## Описание проекта

Flexo Print Website — это веб-приложение, разработанное для компании,
занимающейся флексографической печатью.

Цель проекта — создать современный веб-сайт, который позволит компании
представлять свои услуги, демонстрировать портфолио работ и принимать
заявки от клиентов через интернет.

Веб-сайт предназначен для повышения удобства взаимодействия с клиентами,
расширения клиентской базы и улучшения онлайн-присутствия компании.

## Архитектура проекта

В проекте используется **многослойная архитектура (Layered Architecture)**.

Система разделена на несколько уровней:

### Frontend

Отвечает за пользовательский интерфейс и взаимодействие с пользователем.

### Backend

Обрабатывает запросы пользователей и реализует бизнес-логику приложения.

### Database

Отвечает за хранение данных приложения.

Взаимодействие компонентов:
Пользователь → Frontend → Backend → Database

## Макет пользовательского интерфейса

Макет пользовательского интерфейса приложения был разработан в инструменте **Figma**.  
Он используется для визуального проектирования будущего веб-сайта компании **Артфлекс22**, занимающейся флексографической печатью.

### Ссылка на макет

https://www.figma.com/design/AKT04wWt25YGEQHxqabYrW/Untitled?node-id=0-1&t=WbzkaWiAye1Ip0jQ-1

### Назначение макета

Макет предназначен для предварительного проектирования пользовательского интерфейса веб-сайта.  
Он позволяет визуально представить структуру сайта, расположение элементов интерфейса и основные пользовательские сценарии взаимодействия с системой.

Использование макета позволяет:

- определить структуру страниц сайта;
- продумать удобную навигацию;
- спроектировать пользовательский интерфейс до начала полноценной разработки.

### Реализованные экраны

В макете разработано 5 основных экранов:

1. **Главная страница**  
   Содержит информацию о компании, преимуществах и основные кнопки навигации.

2. **Страница услуг**  
   Отображает перечень услуг компании в области флексографической печати.

3. **Страница портфолио**  
   Демонстрирует примеры выполненных работ и продукции.

4. **Страница оформления заявки**  
   Содержит форму для отправки заявки на печать продукции.

5. **Страница контактов**  
   Содержит контактную информацию компании и карту расположения.

### Пользовательские сценарии

Макет отражает следующие основные пользовательские сценарии:

- пользователь заходит на сайт и знакомится с информацией о компании;
- пользователь изучает перечень услуг;
- пользователь просматривает портфолио выполненных работ;
- пользователь заполняет форму заявки для заказа печати;
- пользователь получает контактную информацию компании.

Макет является основой для дальнейшей реализации пользовательского интерфейса веб-приложения.
